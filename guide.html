Hearth Cmd Setup Guide

Hearth Cmd is mission control for your AI coding agents. You install a small CLI on each machine where you want agents to run, log in once with your email, and start the daemon. From then on, you spawn and manage agents from your phone. The CLI's permission engine intercepts every read, write, network call, and shell command and routes them to the app for approval. Smart rules auto-approve trusted actions so your agents keep working.

Supported Agents

  • Claude Code
  • GitHub Copilot CLI
  • Cursor CLI
  • Codex CLI
  • Pi

What You'll Need

  • One of the supported agents installed and working
  • An iPhone with the Hearth Cmd app installed (iOS App Store listing coming soon)
  • macOS, Linux, or WSL2

Step 1: Install the CLI

curl -sSL https://hearthcmd.com/install.sh | bash

Downloads the latest binary for your system and installs it to /usr/local/bin/hearth.

Step 2: Log In

Run hearth login with your email:

hearth login you@example.com

Hearth Cmd emails you a 6-digit code. Check your inbox, copy the code, and paste it at the prompt:

A 6-digit code has been sent to you@example.com.
Enter the code: 123456

If this is your first time logging in, you'll also be asked for a display name and an organization name (press Enter to accept the suggested defaults). Once verified, this machine is enrolled as a host under your account — you don't need to log in again on this machine unless you sign out.

Step 3: Start the Daemon

hearth start

The daemon runs in the background and listens for spawn requests from the app. You can check on it any time with:

hearth status

Stop it with hearth stop. It does not auto-start on boot — run hearth start after a reboot if you want the host available to your phone.

Step 4: Sign In on Your Phone

Open Hearth Cmd on your iPhone, enter the same email address, and enter the 6-digit code that arrives in your inbox. The app picks up the host you just enrolled and shows it in the host list.

Step 5: Spawn an Agent from the App

Everything from here happens on your phone. To start a new agent session:

  1. Tap + New Agent on the home screen.
  2. Pick the host you want the agent to run on.
  3. Pick a working directory (or add a new one and point it at any folder on that host).
  4. Pick the agent — Claude Code, Copilot CLI, Cursor CLI, Codex CLI, or Pi.
  5. Tap Start.

The agent launches on your laptop in yolo mode — it never blocks on a terminal prompt. The app shows the live activity stream as the agent works. The first time the agent does anything that needs permission (writing a file, running a command, hitting the network), you'll get a prompt on your phone.

Day-to-Day from the App

  • Approve or deny requests. Each request shows the tool, the input, and the risk level. Tap Allow, Always Allow, Deny, or Deny & Stop.
  • Watch the activity stream. Every tool call, file edit, and response streams in real time. Scrub back through anything the agent has done in the session.
  • Talk to your agent. The Activity tab has an input bar. Anything you type is sent to the agent as if you typed it at the terminal.
  • Run multiple agents at once. Spawn agents on different working directories, or different hosts, and switch between them in the home screen.
  • Pull the plug. Tap the stop button on any session to terminate the agent immediately.

CLI Reference

Command Description
hearth login <email> Log in via email OTP and enroll this host. --invite <token> joins an existing org.
hearth start Start the daemon on this host.
hearth stop Stop the daemon.
hearth status Show daemon status.
hearth talk Open a TUI to talk to active agent instances from this terminal.
hearth hh Manage household, users, hosts, devices, agents, and invites.
hearth wd Manage working directories (list, create, update, abandon, delete).
hearth update Update hearth to the latest version.
hearth logout Sign out on this machine: stops the daemon, revokes credentials, clears config.

Run hearth <command> --help for details. Config lives in ~/.hearth/config.

How the Rules Work

Rules are the core of the system. Tapping "Always Allow" doesn't just approve the request — it generates a pattern that auto-approves future matching commands without interrupting you.

Pattern generation

Patterns are subcommand-aware. The system knows that git push and git status are different operations:

  • go build ./cmd/foo → pattern Bash(go build **) — matches any go build
  • git push origin main → pattern Bash(git push **) — matches any git push, but not git reset
  • aws s3 cp file.txt s3://bucket/ → pattern Bash(aws s3 cp **) — three-word prefix for AWS/GCloud

Destructive command safety

Commands like rm, sudo, kill, and chmod use exact matching. Granting rm -rf /tmp/coverage creates a rule that matches only that exact string — not rm -rf /home, not rm -rf /tmp/coverage /etc.

Compound commands

When the agent runs git add . && git commit -m "msg" && git push, the system splits it into three segments and generates a separate pattern for each. All segments must have matching rules for the compound command to auto-approve. If you've granted git add and git commit but not git push, the compound command still comes to your phone.

In the app, each segment is color-coded by risk (green/orange/red) and segments already covered by rules show a checkmark overlay — so you can see at a glance which part of the command is the one that needs your attention.

File operations

For Read, Write, and Edit tools, patterns match on the directory. Granting an edit to /home/user/project/src/main.go creates the pattern Edit(/home/user/project/src/**), which covers future edits in that directory.

Talk to Your Agent from Your Phone

The Activity tab in the app shows an input bar at the bottom. Anything you type is sent directly to your agent session as if you typed it at the terminal.

Use this to:

  • Answer prompts when your agent asks a question
  • Provide additional context or instructions mid-session
  • Nudge the agent in a different direction
  • Respond to interactive dialogs

Talk to Your Agent requires Hearth Cmd Pro. Pro also includes push notifications so you never miss a request.

Tips

  • The first few sessions generate the most requests. Once you've built up rules for your common patterns, interruptions drop off quickly.
  • Check the Grants tab to see your rules grouped by project. Swipe left to delete any rule.
  • If you're in a Focus mode on iOS, make sure Time Sensitive notifications are allowed for Hearth Cmd — permission requests use this priority.
