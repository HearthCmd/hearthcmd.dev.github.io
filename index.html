How It Works

1

Launch your agents

Install hearth-cmd-cli and run hearth connect. Your agent launches in yolo mode so it never blocks on a terminal prompt. Hearth Cmd's permission engine intercepts all reads, writes, network IO, and shell commands — forwarding them to your phone for approval.

2

Build rules, stop blocking

Tap "Always Allow" and Hearth Cmd generates smart rules that auto-approve matching actions. After a few sessions, most requests never reach you. Your agents just keep working.

3

Manage everything from your phone

Run multiple agents across projects. Stream activity, handle approvals, send input, and pull the plug — all from one place.

Features

Smart Rules

Set up rules once, your agents auto-approve trusted actions forever. Subcommand-aware patterns keep things safe — destructive commands use exact matching.

Multi-Agent, Multi-Project

Manage all your agents from one place. Run agents across projects simultaneously — each with its own rules and activity stream.

Push Notifications

Get notified when an agent needs you, even when the app is closed. Time Sensitive alerts break through Focus modes. Hearth Cmd Pro.

Live Activity Stream

See exactly what every agent is doing, across every project. Every tool call, file edit, and decision streams through in real time.

Talk to Your Agent

Send text directly to any agent session from your phone. Answer prompts, provide context, or redirect — no laptop needed. Hearth Cmd Pro.

Pull the Plug

Remotely terminate any running agent, instantly. One tap from anywhere.

Command Risk Intelligence

Commands are color-coded by risk level using a registry of 170+ CLI tools. See at a glance which parts of a compound command need attention.

Rich Previews

See syntax-highlighted commands, file diffs, and tool-specific views before you approve.

Session Registration

Every agent session must be approved on your phone before it can send requests. You control which sessions reach you.
