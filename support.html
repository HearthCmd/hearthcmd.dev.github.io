Troubleshooting

Host not appearing in the app

  • Make sure the daemon is running: hearth status.
  • Confirm you're signed in to the same account on the CLI and the app — hearth login emails an OTP to the address you specify, and the app's email must match.
  • Check the About tab in the app — status should show "Connected" with a green dot.

Agent won't spawn from the app

  • Confirm hearth status shows the daemon as running on the target host.
  • Make sure the working directory exists on that host and is readable.
  • Make sure the agent binary you picked (e.g. claude) is installed and on the daemon's PATH.
  • Check the daemon log for errors. Path is shown by hearth status.

Talk to Your Agent not working

  • Talk to Your Agent requires a Hearth Cmd Pro subscription.
  • The agent must have been spawned via the app (or via the daemon on this host).
  • Check that the Activity tab shows an input bar at the bottom when viewing the session.

Not receiving push notifications

  • Check that notifications are enabled: Settings → Hearth Cmd → Notifications.
  • If using a Focus mode, make sure Hearth Cmd is in the allowed apps list or Time Sensitive notifications are enabled.
  • Push notifications are sent when the app is backgrounded. If the app is in the foreground, requests arrive directly via WebSocket.

Requests timing out

  • The default timeout is 590 seconds.
  • If the app is not connected and push notifications fail, the request will time out and be denied.
